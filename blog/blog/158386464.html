Windows到Linux应用程序远程部署方案

1、 环境描述

Windows XP、Windows XP上的VMware Workstation虚拟机、虚拟机上安装的CentOS5.0

2、 从Windows XP复制文件到CentOS5.0的方法

1） ftp方法：Windows XP自带ftp客户端程序；CentOS5.0上部署ftp服务器程序。

2）      Windows XP：

putty(SSH的Windows客户端，用于与CentOS5.0系统建立连接)；

     putty自带的pscp、psftp、plink提供命令行的Windows XP和CentOS5.0文件互相复制；

winscp可视化界面的Windows XP和CentOS5.0文件互相复制软件；

CentOS5.0：

自带SSH服务器程序。

3、 示意图

注意事项：

1）pscp、psftp、pslink（基于SSH）也可以单独下载，在cmd中运行时，需要设置环境变量PATH，使其包含pscp、psftp、plink可执行文件的目录。在cmd输入pscp，显示其用法，说明配置成功，psftp、plink相同。

2）使用putty或telnet登陆到CentOS上时，就是在Linux环境下，不能使用Windows下的命令。

3）如果在Windows XP上部署了SSH服务器程序，那么在putty登陆到CentOS5.0后，可以使用scp（基于SSH）进行Windows XP和CentOS5.0文件互相复制。

4）如果CentOS5.0配置了ftp服务器，Windows XP端可以通过ftp上传或下载CentOS5.0上的文件。

5）不同的Linux之间copy文件常用有3种方法 ：


第一种就是ftp，也就是其中一台Linux安装ftp Server，这样可以另外一台使用ftp的client程序来进行文件的copy；


第二种方法就是采用samba服务，类似Windows 文件copy的方式来操作，比较简洁方便；


第三种就是利用scp命令来进行文件复制。


       scp是有Security的文件copy，基于ssh登录。操作起来比较方便，比如要把当前一个文件copy到远程另外一台主机上，可以如下命令。
       scp /home/daisy/full.tar.gz root@172.19.2.75:/home/root
       然后会提示你输入另外那台172.19.2.75主机的root用户的登录密码，接着就开始cp和ungzip了
如果想反过来操作，把文件从远程主机copy到当前系统，也很简单如下：
       scp root@172.19.2.75:/home/root /home/daisy/full.tar.gz

4、pscp、psftp命令的具体使用

1）pscp在cmd中的命令行，实现把Windows XP上的文件复制到远程CentOS5.0上，具体命令:

pscp e:\SMSAPI.jar root@192.168.150.128:/telnetTest

root@192.168.150.128’s password:******

结果：把Windows XP上E盘根目录下的文件SMSAPI.jar，复制到了CentOS5.0上/telnetTest目录下。

2）psftp在cmd中的命令行，实现把Windows XP上的文件复制到远程CentOS5.0上，具体命令:

psftp

open 192.168.150.128

login as：root

root@192.168.150.128’s password:******

cd /telnetTest (改变CentOS5.0的当前目录)

lcd e:\（改变Windows XP的当前目录）

put SMSAPI.jar

结果：把Windows XP上E盘根目录下的文件SMSAPI.jar，复制到了CentOS5.0上/telnetTest目录下。

5、putty远程登陆CentOS5.0系统

     1）直接运行

         cd /telnetTest （进入SMSAPI.jar所在目录）

          java -jar SMSAPI.jar(运行程序)

     2）把运行SMSAPI.jar的命令编辑为shell可执行文件sms

        cd /telnetTest

       vi sms(按i键，进入输入模式，输入：#！/bin/bash

                                                                cd /telnetTest

                                                                java -jar SMSAPI.jar

                  :wq(保存退出))

      chmod +x sms（改成可执行文件）

      ./sms(运行程序)

     3）把sms文件所在的目录，设置在环境变量的PATH中

     vi /etc/profile(打开配置文件所在的文件)

     在最后一行加上：export PATH="$PATH:/telnetTest"

     :wq(保存退出)

      reboot(重启CentOS5.0，使配置文件生效)

      sms（在任何目录下，直接输入sms就可运行SMSAIPI.jar）

至此远程部署完毕。
