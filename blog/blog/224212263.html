公司IT系统中遇到的问题总结

1、IIS6或者Tomcat假死问题

问题描述：按下“F5”键，刷新调用Sqlserver2005数据库的页面，刷新N（大约10次）次，页面无法显示，IIS6假死。

原因：数据库的链接没有释放。要在代码中强制关闭数据库链接。

2、JS文件通过SSL VPN下载速度慢的问题

问题描述：页面中引用Jquery.js，大约70k，通过ssl VPN访问，下载Jquery.js文件需要很长的时间，原因是SSL VPN对Jquery.js进行了处理，导致，页面加载像蜗牛一样慢。

解决方案：把Jquery.js文件放到SSL VPN之外的公网上（例如：http://211.111.1.111/js/Jquery.js），使Jquery.js的加载不通过SSL VPN，这样Jquery.js的加载速度很快，但是安全性稍差。

VPN过滤使用：http://*Jquery.js

http://211.111.1.111/js/Jquery.js经过SSL VPN的处理后的格式：https://vpn的IP地址/+CSCO+kh3231312E3131312E312E313131/js/Jquery.js

安全和不安全内容或混合内容表示某个网页试图同时使用安全 (HTTPS/SSL) 和不安全 (HTTP) Web 服务器连接来显示元素。这通常在显示来自不安全的服务器的图像、横幅或脚本的在线商店或金融站点上出现。

显示混合内容的风险在于，不安全网页或脚本可能能够访问安全内容中的信息。

Internet Explorer 使用一种称为安全套接字层 (SSL) 的加密协议访问安全网页。这些页面使用 HTTPS 前缀，而普通网页使用 HTTP 前缀。
