在压力测试中，多线程和压力的关系？

    在我的理解中，启动线程数和对系统的压力是成正比的，线程越多对系统的压力越大。现在发现不是这样的，比如如果一个线程不间断的运行，那么CPU的利用率是100%，是全压测试。当然这是一种特殊的情况，多线程一般情况下能够提高CPU的利用率，提高系统的效率。

     在多线程的运行中，我们最关注的是线程的对临界资源的并发使用问题。在java中利用synchronized关键字实现，具体用法：

1、同步对象，用法：

synchronized（对象）{

       临界资源

}

2、同步方法

public synchronized void method（）{}：加对象锁

public static synchronized void method（）{}：加类锁

在JDK中的包java.util.concurrent，包括很多并发编程的常用工具。
